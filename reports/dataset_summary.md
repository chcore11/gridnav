# 专家轨迹数据集摘要

## 数据来源

当前数据集由 A* 在 100 张随机 `10 x 10` GridWorld 地图上搜索并生成。只有成功找到路径的地图会转换成 state-action 样本。

## 参数设置

- 地图大小：`10 x 10`
- 障碍物比例：`0.2`
- 起点和终点：每张地图随机采样，且不相同
- 起点和终点最小曼哈顿距离：`6`
- 随机种子：`42`
- `map_id`：原始尝试地图编号，从 `0` 开始；无路径地图的编号会被跳过。
- `step_id`：一条成功路径中的动作序号，从 `0` 开始。

## 生成统计

- 尝试地图数量：100
- 成功找到路径地图数量：98
- 无路径地图数量：2
- 总样本数量：970
- 平均路径移动步数：9.90

## Action 分布

| action | 方向 | 样本数量 | 占比 |
| --- | --- | ---: | ---: |
| `0` | up | 226 | 23.30% |
| `1` | down | 269 | 27.73% |
| `2` | left | 275 | 28.35% |
| `3` | right | 200 | 20.62% |

四个方向都获得了较多样本，分布比固定左上角起点和右下角终点时更合理。当前各动作仍不是完全相等，但已不再几乎只包含 down 和 right。

## 字段含义

- `map_id`：原始尝试地图编号。
- `step_id`：当前动作在该路径中的序号。
- `agent_x`、`agent_y`：智能体当前坐标。
- `goal_x`、`goal_y`：目标点坐标。
- `obs_up`、`obs_down`、`obs_left`、`obs_right`：对应方向越界或存在障碍物时为 `1`，否则为 `0`。
- `action`：专家动作，`0 = up`、`1 = down`、`2 = left`、`3 = right`。

## 当前局限

- 仍然是二维简化 GridWorld，和真实导航环境有较大差距。
- 状态特征只包含当前位置、目标和局部障碍，看不到完整地图。
- 该数据集已用于传统机器学习 baseline，按 `map_id` 划分训练集和测试集。
- 当前 baseline 学习的是 `state -> action` 单步动作预测，没有进行完整导航 rollout。
- 随机地图数量和障碍物比例后续可以继续调整。
