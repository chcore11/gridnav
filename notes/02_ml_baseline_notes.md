# 机器学习 Baseline 学习笔记

## 我现在怎么理解特征和标签

这次任务是根据当前 state 预测 A* 专家的下一步 action。模型看到的输入叫特征，当前一共有八个：

- `agent_x`、`agent_y`：智能体当前坐标。
- `goal_x`、`goal_y`：目标坐标。
- `obs_up`、`obs_down`、`obs_left`、`obs_right`：当前点四周是否越界或有障碍。

我把特征理解成模型做判断时能看到的信息。A* 专家实际选择的 `action` 是标签，也就是希望模型学会预测的正确答案。动作使用 `0 = up`、`1 = down`、`2 = left`、`3 = right` 编码。

## 为什么这是分类任务

模型需要从 up、down、left、right 四个离散选项中选一个，而不是预测一个可以任意取值的连续数字。虽然 action 在 CSV 中保存成 `0` 到 `3`，这些数字只是类别编码，不表示 right 比 up 大三倍，所以这是四分类任务，不是回归任务。

## 为什么需要训练集和测试集

训练集用来让模型学习 state 到 action 的规律，测试集用来检查模型面对没参与训练的数据时表现怎么样。如果使用训练数据评价，模型可能只是记住已有样本，得到的 accuracy 不能说明它是否有泛化能力。

这次不是随机按 CSV 的行划分，而是先按 `map_id` 划分地图，再取出对应的所有路径样本。因为同一条 A* 路径中的相邻 state 很相似，如果随机按行划分，同一张地图甚至同一条路径的样本可能同时进入训练集和测试集，造成数据泄漏，测试结果会显得过于乐观。按 `map_id` 划分后，一张地图只属于训练集或测试集中的一边，测试更接近“面对新地图”的情况。

## 三个 baseline 模型的区别

### KNN

KNN 会在训练数据中寻找和当前特征最接近的几个样本，再根据这些邻居的 action 投票。它比较直观，几乎没有真正的训练过程，但预测时需要查找邻居，而且距离会受特征尺度影响，所以代码中先做了标准化。

### Logistic Regression

Logistic Regression 会学习每个特征对不同 action 的线性权重，再输出四个类别的概率。它比 KNN 更像是在总结一条全局规则，但线性决策边界可能难以表达复杂的避障选择。它同样先做了标准化。

### Decision Tree

Decision Tree 会学习一连串类似“目标是否在左边”“下方是否有障碍”的 if-else 式条件分支。它不要求特征标准化，也比较容易解释，但如果树切分得太细，可能会把训练集里的偶然细节当成普遍规律，这就是一种过拟合。

## 这次结果怎么理解

本次使用固定随机种子 `42`，按 `map_id` 将约 80% 地图作为训练集、20% 地图作为测试集。三个模型的详细 accuracy、classification report 和 confusion matrix 记录在 `reports/ml_baseline_result.md`，图保存在 `figures/ml_baseline_confusion_matrix.png`。

实际划分得到 78 张训练地图和 20 张测试地图，对应 776 条训练样本和 194 条测试样本。三个模型的测试集 action accuracy 是：

- KNN：`0.8144`
- Logistic Regression：`0.8557`
- Decision Tree：`0.7732`

这次 Logistic Regression 表现最好。我原本会觉得 Decision Tree 的条件分支更适合处理避障，但默认参数下它的测试 accuracy 反而最低。这说明直觉不能代替测试结果，也可能表示树在训练样本上学得太细，面对新地图时泛化较差。

我目前会重点同时看三个内容：

- Accuracy：像整体体检分数，表示总共有多少测试样本预测正确。
- Confusion matrix：像详细体检报告，可以直接查看每个真实动作具体容易错成哪个动作。
- Classification report：像每个动作的单项诊断，分别展示 precision、recall 和 F1-score。

当前特征只有局部障碍，没有完整地图信息。两个样本即使八个特征相同，也可能因为远处障碍和 A* 路径不同而需要不同动作。因此模型不一定能只靠当前特征完全复现 A*。

## 为什么 baseline 是后续行为克隆的参照

传统机器学习 baseline 先给出一组简单模型能够达到的结果。后续进入 PyTorch 行为克隆时，神经网络也会学习同一个 `state -> action` 任务，所以可以和 baseline 使用相同的数据划分与指标比较。

如果神经网络没有超过简单 baseline，就需要检查网络、训练过程或特征设计，而不能只因为使用了 PyTorch 就认为模型更好。baseline 也让我先把数据、分类和评估流程理解清楚，再进入更复杂的训练代码。

## 当前阶段的边界

Action prediction 是拿测试集中的一个 state，让模型预测这一步应该采取哪个 action。Rollout 则是让模型从起点开始连续选择动作，并检查最终是否到达目标。

这次只做单步 action prediction，没有做完整导航 rollout。单步 action accuracy 高不代表导航一定成功，因为一次错误可能让智能体进入专家数据中很少出现的状态，后面的错误还可能继续累积。因此当前阶段只能说 Logistic Regression 的单步动作预测表现最好，不能说模型已经会导航。
