# 专家轨迹数据学习笔记

## 什么是专家轨迹

专家轨迹可以理解为一个表现比较好的规划器或智能体完成任务时留下的过程记录。它不只包含最后结果，还包含每一步所在的状态，以及在这个状态下采取的动作。

当前项目里，A* 知道完整地图，并且能搜索出一条合理的最短路径，所以可以先把 A* 当作“专家”，用它的路径作为专家演示。

## 从 path 拆成 state-action

A* 返回的 path 是一个坐标列表，例如：

```text
(0, 0) -> (0, 1) -> (0, 2)
```

生成数据时会遍历每一对相邻坐标，把 `current` 作为 state，再根据 `current -> nxt` 的坐标变化得到 action。上面的例子可以拆成两条样本：

```text
state=(0, 0), action=down
state=(0, 1), action=down
```

最后一个 goal 点没有下一步动作，所以不会单独生成样本。因此，样本数量等于路径点数量减一。

## state 特征

当前每个 state 使用八个特征：

- `agent_x`、`agent_y`：智能体当前所在位置。
- `goal_x`、`goal_y`：目标点所在位置。
- `obs_up`：上方越界或有障碍物时为 `1`，否则为 `0`。
- `obs_down`：下方越界或有障碍物时为 `1`，否则为 `0`。
- `obs_left`：左侧越界或有障碍物时为 `1`，否则为 `0`。
- `obs_right`：右侧越界或有障碍物时为 `1`，否则为 `0`。

这些特征同时提供了当前位置、目标位置和附近障碍信息，可以作为后续机器学习模型的输入。

## action 编码

为了让动作能作为机器学习标签保存，四个方向被编码成整数：

- `0 = up`
- `1 = down`
- `2 = left`
- `3 = right`

例如相邻路径点从 `(2, 3)` 走到 `(3, 3)`，`x` 增加了 `1`，所以 action 是 right，对应数字 `3`。

## 当前初版的局限

最开始的专家数据只来自 demo 中的一张固定地图。这个版本适合先跑通“A* 路径 -> state-action 样本 -> CSV”的流程，但不适合直接拿去训练模型。因为地图不变时，路径和动作也很单一，模型可能只是在记住这一条路线。

## 为什么需要随机多地图

现在脚本会使用固定随机种子生成多张随机地图，再让 A* 分别寻找路径。这样做以后，数据中会出现更多不同的障碍布局、当前位置和路径选择。

我目前对“泛化”的理解是：模型不只是会处理训练时见过的路线，而是面对新的地图或状态时，也能根据学到的规律选择合理动作。随机多地图不能直接保证模型一定能泛化，但它比单张固定地图更适合为泛化做准备。

固定随机种子也很重要。它让每次运行得到相同的一组地图和统计结果，方便检查代码修改前后有没有意外变化。

## 为什么检查 action 分布

生成数据后需要检查 action 分布，也就是 up、down、left、right 各有多少样本。如果某个动作特别多，其他动作特别少，后续模型可能更容易一直猜数量最多的动作，而不是真的理解状态。

之前起点固定为左上角 `(0, 0)`，终点固定为右下角 `(9, 9)`。从起点到终点的主要方向天然就是 down 和 right，只有障碍物迫使路径绕路时才偶尔需要 up 或 left。因此旧数据出现了 up 1、down 757、left 1、right 757 的严重偏差。

现在每张地图都会随机生成 start 和 goal，并要求两者的曼哈顿距离至少为 6。目标可能位于起点的上、下、左、右等不同方向，所以 A* 路径中的四种动作会更多样。这样做不保证动作数量完全相等，但可以避免方向几乎被固定死。

动作分布会影响下一阶段的机器学习 baseline。如果数据严重偏向 down 和 right，模型即使主要预测这两个动作也可能得到看起来不错的准确率，却没有真正学会根据状态选择方向。更有多样性的动作样本能让后续评估更有意义。

## 为什么无路径地图要跳过

随机放置障碍物以后，有些地图可能把起点和终点完全隔开。A* 在这种地图上找不到路径，也就没有可以转换的专家动作。

这类地图不能强行生成 state-action 样本，所以当前阶段会记录失败数量并跳过。以后如果研究“无法到达”的判断，也许可以把它们作为另一类数据，但那不是这次专家轨迹生成的目标。

## 和下一阶段的关系

这个阶段仍然只负责生成和检查专家数据，还没有进入机器学习 baseline。下一阶段才会读取这些 state-action 样本，划分训练集和测试集，再训练一个简单模型预测 action。

所以在进入下一阶段之前，需要先确认数据字段、路径转换、动作编码、样本数量和动作分布都合理。否则模型结果不好时，很难判断问题来自模型还是来自数据。

## 当前多地图版本的局限

当前版本虽然扩展到了随机多地图和随机 start/goal，但仍然是二维简化 GridWorld。状态只包含当前位置、目标位置和四个方向的局部障碍，没有完整地图信息，也还没有划分训练集与测试集。地图数量、障碍物比例和起终点距离要求后续都可以继续调整。
