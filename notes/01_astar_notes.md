# A* 路径规划笔记

## 当前阶段目标

当前阶段主要是先理解 GridWorld 环境怎么表示地图，以及 A* 怎么在已知地图中规划路径。这里还不是在训练机器学习模型，而是在搭好后面的基础：先用 A* 生成专家数据，再尝试机器学习 baseline、行为克隆和强化学习。

我觉得先把环境、坐标和路径搜索弄清楚很重要，不然后面即使开始训练模型，也很难判断模型输出的动作是否正确。

## GridWorld 如何表示地图

`env/grid_world.py` 中的 `GridWorld` 使用 `(x, y)` 坐标表示地图中的位置：

- `x` 表示列，向右增大；
- `y` 表示行，向下增大；
- 因此左上角是 `(0, 0)`。

这里需要特别注意，坐标写法是 `(x, y)`，但 numpy 数组的访问顺序是先行后列，所以访问数组时要写成 `grid[y, x]`。例如 `to_array()` 中保存障碍物时使用：

```python
grid[y, x] = 1
```

`GridWorld` 会分别保存地图宽度 `width`、高度 `height`、起点 `start`、终点 `goal` 和障碍物 `obstacles`。起点和终点都是一个坐标元组，例如 demo 中的起点是 `(0, 0)`，目标点是 `(9, 9)`。障碍物则是一组坐标，构造环境时会被转成 `set`。

障碍物使用 `set` 的好处是判断某个位置是不是障碍物比较直接，而且查找速度通常很快。代码中的 `passable(node)` 只需要判断：

```python
node not in self.obstacles
```

`to_array()` 还会把地图转换成 numpy 数组，用不同数字表示不同内容：普通空地是 `0`，障碍物是 `1`，路径是 `2`，起点是 `3`，目标点是 `4`。

## neighbors() 的作用

`neighbors(node)` 的作用是返回当前节点上下左右所有可以通行的位置。它先根据当前坐标生成四个候选位置：

- `(x + 1, y)` 是 right；
- `(x - 1, y)` 是 left；
- `(x, y + 1)` 是 down；
- `(x, y - 1)` 是 up。

生成候选位置之后，`neighbors()` 会用 `in_bounds()` 过滤掉超出地图边界的位置，再用 `passable()` 过滤掉障碍物。因此，最后返回的节点都是下一步实际可以到达的位置。

当前代码还没有把 action 显式编码成整数或字符串，例如没有定义 `0 = up`、`1 = right`。不过四个动作已经隐含在相邻坐标的变化中了。之后生成专家数据时，可以根据前后两个坐标的差值，把路径转换成明确的 action。

## A* 的核心公式 f(n)=g(n)+h(n)

A* 的核心公式是：

```text
f(n) = g(n) + h(n)
```

我目前的理解是：

- `g(n)` 是从起点走到当前节点 `n` 的真实代价；
- `h(n)` 是从当前节点 `n` 到目标点的估计代价；
- `f(n)` 是两者相加后的综合优先级，A* 会优先搜索 `f(n)` 较小的节点。

在 `planning_astar/astar.py` 中，每次上下左右移动的代价都是 `1`，所以：

```python
new_cost = cost_so_far[current] + 1
```

代码里的 `cost_so_far[nxt]` 对应 `g(n)`，也就是从起点到 `nxt` 的当前最小真实代价。`manhattan(nxt, goal)` 对应 `h(n)`，用来估计 `nxt` 到目标点还需要多远。下面这一行就是在计算 `f(n)`：

```python
priority = new_cost + manhattan(nxt, goal)
```

本项目使用的启发式函数是曼哈顿距离：

```python
abs(ax - bx) + abs(ay - by)
```

曼哈顿距离适合当前 GridWorld，因为智能体只能上下左右移动，不能斜着走。它相当于先忽略障碍物，估计横向和纵向总共还需要走多少步。

## frontier / cost_so_far / came_from 分别是什么

`frontier` 是待搜索节点的集合，也可以理解为 A* 中的 open set。代码使用 `heapq` 管理 `frontier`，放入其中的是 `(priority, node)`。由于堆会优先弹出 priority 较小的元素，所以 A* 能先搜索当前看起来更有希望的节点。

`cost_so_far` 记录从起点到某个节点的当前最小真实代价，也就是保存各节点的 `g(n)`。如果之后找到一条到达同一节点但代价更小的路线，就会更新这个值。

`came_from` 记录每个节点是从哪个节点到达的，可以把它理解为父节点关系。例如 `came_from[nxt] = current` 表示当前已知的较优路线中，`nxt` 的前一个节点是 `current`。它主要用于搜索结束后恢复完整路径。

## 路径是如何恢复的

A* 搜索过程中并不会一直给每个候选节点保存一条完整路径，这样会产生很多重复内容。当前实现只通过 `came_from` 保存每个节点的父节点。

找到 goal 后，代码从 goal 开始，根据 `came_from[current]` 一路反向追踪，直到起点的父节点 `None`：

```python
while current is not None:
    path.append(current)
    current = came_from[current]
```

这时得到的路径顺序是从 goal 到 start，所以最后需要执行 `path.reverse()`，才能得到从 start 到 goal 的正常路径。

如果搜索结束后 goal 不在 `came_from` 中，说明没有找到可行路径，函数会返回 `None`。

## 当前 demo 的运行结果理解

`scripts/run_astar_demo.py` 创建了一张固定的 `10 x 10` 地图，起点是 `(0, 0)`，目标点是 `(9, 9)`，中间设置了两部分障碍物。

当前运行结果中的 `Path length: 19` 表示返回的路径列表里共有 19 个坐标点，其中包括起点和终点。因此实际移动步数不是 19，而是：

```text
19 - 1 = 18
```

从当前路径来看，智能体大致先沿左侧向下走，再沿底部向右走到目标点。因为从 `(0, 0)` 到 `(9, 9)` 至少需要向右走 9 步、向下走 9 步，所以 18 步也是这张地图上的合理最短步数。

demo 会调用 `world.to_array(path)` 把障碍物、路径、起点和终点转换成数组，再使用 matplotlib 绘图。生成的图片保存到：

```text
figures/astar_demo.png
```

## A* 为什么不是机器学习

A* 是一种搜索算法，不是机器学习方法。它不需要训练数据，不会计算 loss 并更新模型参数，也不会通过过去的经验自动学习更好的策略。

它做的事情是在地图已知的情况下，按照搜索规则和启发式函数寻找路径。只要给定同样的地图、起点和终点，当前实现就会按相同规则进行搜索。

## A* 为什么可以生成专家数据

虽然 A* 本身不是机器学习，但它可以在已知地图中生成一条合理路径，因此可以充当一个专家规划器。

后续可以把 A* 路径中相邻的坐标拆成多个 `state -> action` 样本。例如从 `(0, 0)` 到 `(0, 1)` 可以转换成当前状态对应 `down` 动作。收集很多地图上的这类样本后，就可以组成专家轨迹数据集，用于训练机器学习 baseline 或行为克隆模型。

所以 A* 在这个项目中的一个重要作用，是为后续模型提供可学习的正确示范。

## 当前代码的不足

1. 当前 `GridWorld` 还没有 `reset()` 和 `step(action)`，因此还不是一个可以直接用于强化学习交互的完整环境。
2. 当前 action 还没有显式编码，四个动作只隐含在坐标变化中。
3. 当前 demo 只有一张固定地图，还不能反映算法在不同地图上的表现。
4. 后续需要随机生成多张地图，并使用 A* 为这些地图生成专家轨迹数据集。

## 当前阶段小结

用我自己的话总结，GridWorld 负责表示地图、边界、障碍物、起点和终点，A* 负责在这张已知地图中搜索路径。A* 最核心的是 `f(n)=g(n)+h(n)`：`g(n)` 是已经走过的真实代价，`h(n)` 是到目标点的估计代价。

这个阶段的意义不是直接做机器学习，而是先把环境和路径规划逻辑理解清楚，并为后续生成专家数据、训练行为克隆模型以及继续做强化学习打基础。
