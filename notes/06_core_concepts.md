# 核心概念

## A* 规划

A* 是一种图搜索算法，用来从起点找到通向终点的低代价路径。在 GridWorld 中，每个格子可以看成一个节点，相邻可移动格子之间的移动可以看成边。

A* 的核心评价函数是：

```text
f(n) = g(n) + h(n)
```

- g(n)：从起点走到当前节点 n 已经花费的实际代价。
- h(n)：从当前节点 n 到终点的启发式估计代价。
- f(n)：综合考虑已经花费的代价和预计还要花费的代价。

如果 h(n) 设计合理，例如在四方向网格中使用曼哈顿距离，A* 通常可以比普通广度优先搜索更快地找到路径。这个项目会先用 A* 生成可解释的专家路径，再把这些路径作为后续学习方法的数据来源。

## 机器学习 Baseline

机器学习 baseline 是一个简单但重要的对照方法。它不追求最强性能，而是帮助确认任务、数据和评估流程是否合理。

在本项目中，可以把 A* 生成的专家轨迹转成监督学习数据：

- 特征：描述当前状态的信息，例如当前位置、目标位置、障碍物信息、相对方向或局部网格。
- 标签：专家在该状态下选择的动作，例如上、下、左、右。
- 训练集：用于让模型学习特征到动作的映射。
- 测试集：用于检查模型在未见过样本上的表现。
- 泛化：模型能否在新地图、新起点或新终点上做出合理动作。
- baseline：一个基础模型，例如逻辑回归、决策树或随机森林，用来作为后续复杂模型的比较对象。

如果 baseline 表现很差，需要先检查特征、标签、数据划分和评估方式，而不是马上增加模型复杂度。

## 行为克隆

行为克隆是模仿学习的一种基础方法。它把专家的行为当作监督学习数据：输入是状态，目标输出是专家动作。

它之所以是监督学习，是因为训练时已经知道正确答案，也就是专家在每个状态下选择的动作。模型不需要自己探索奖励，而是通过最小化预测动作和专家动作之间的误差来学习策略。

在 GridNav 中，A* 负责提供专家轨迹，行为克隆模型负责学习这些轨迹背后的动作选择规律。需要注意的是，行为克隆只学习专家数据分布中的状态，如果模型走到专家很少出现的状态，可能会连续犯错。

## PyTorch 训练流程

PyTorch 训练循环通常包括以下步骤：

- forward：把输入数据传给模型，得到预测结果。
- loss：计算预测结果和真实标签之间的差距。
- backward：根据 loss 反向传播，计算参数梯度。
- optimizer.step：优化器根据梯度更新模型参数。

一个基本训练循环的重点不是语法本身，而是理解数据如何流过模型、误差如何被计算、梯度如何被传播，以及参数为什么会逐步改变。

在行为克隆任务中，模型输入 GridWorld 状态，输出动作类别。loss 可以衡量模型预测动作和专家动作之间的差异。

## Q-learning

Q-learning 是一种强化学习方法。它不直接模仿专家，而是让智能体和环境交互，并通过奖励信号学习每个动作的长期价值。

核心概念包括：

- state：智能体当前所处的状态，例如当前位置和目标信息。
- action：智能体可以执行的动作，例如上、下、左、右。
- reward：环境给出的反馈，例如到达目标获得正奖励，撞墙或走远获得负奖励。
- policy：策略，表示在某个状态下选择哪个动作。
- Q(s,a)：在状态 s 下执行动作 a 的价值估计。
- epsilon-greedy：一种探索策略。以较小概率 epsilon 随机探索动作，其余时候选择当前 Q 值最高的动作。

Q-learning 的学习重点是：智能体并不知道专家答案，而是通过不断试错更新 Q(s,a)，逐渐形成更好的策略。

## C++ A*

用 C++ 实现 A* 可以帮助把算法理解推进到更工程化的层面。路径规划在机器人、自动驾驶、移动智能体和游戏 AI 中都是基础问题，而 A* 是很多路径规划方法的入门核心。

C++ A* 会涉及：

- 网格地图的数据表示。
- 节点结构和父节点记录。
- 优先队列管理待扩展节点。
- 代价更新和重复节点处理。
- 路径回溯。
- 编译、运行和调试。

机器人路径规划通常需要在地图中寻找可行路径，同时考虑障碍物、代价和实时性。虽然 GridNav 是简化问题，但它能帮助理解更复杂路径规划系统的基础组成。
